SQL发展

1972年，IBM在System R中实现SQUARE语言。 1974年，Boyce和Chamberlin改进SQUARE为SEQUEL。 1981年，IBM推出SQL/DS，用SQL代替SEQUEL。 1986年ANSI公布第一个SQL标准。 1987年ISO将SQL采纳为国际标准。 ​

BETWEEN谓词

BETWEEN谓词用于判断某个值是否属于一个指定的区间
一般形式为:
E [NOT] BETWEEN E1 AND E2
其语义为:
[NOT] (E>=E1 AND E<=E2) E、E1和E2都是表达式，且E1<E2。

  • 例：查询出生时间在1998-01-01到1999-12-31之间的学生的姓名和出生时间

LIKE谓词

LIKE谓词用于部分匹配查询（模糊查询）
一般形式为:

  • “字符串表达式”中通配符:
  • 字符”_”:代表任意一个单字符
  • 字符“%”:代表长度可为零的任意长字符串
    <属性列名> [NOT] LIKE字符串表达式
  • 例：查询姓“王”的所有学生的学号和姓名。 “ \ ” 转义字符:用于去掉一些特殊字符的特定含义，使其被作为普通字符看待, 如用“%”去匹配字符%，用_ 去匹配字符_

IN谓词

IN谓词适用于判断一个值是否属于一个集合
与比较运算符中“=”含义相同。
一般形式为:
E [NOT] IN (V1, V2,…, Vn)
其语义为:
[NOT] (E= V1 OR E= V2 OR… OR E= Vn)

  • 例：查询非数学系和非计算机系的学生的学号、姓名和所在系

NULL谓词

NULL谓词适用于判断属性值是否为空
一般形式为：
<属性列名>IS [NOT] NULL

  • 例：查询所有缺少选课成绩的学生的学号和相应的课程号

语句格式

依次按属性列名1,属性列名2…的值进行分组

    分组过滤： 若要对集合(即分组)进行条件过滤，即满足条件的集合/分组留下，不满足条件的集合/分组剔除。
    Having子句，又称分组过滤子句。需要有Group by子句支持，换句话说，没有Group by子句，便不能有Having子句。
  • 例：查找男生人数超过2名的系的名称。 总括：
  • WHERE 子句从由FROM指定的数据对象中选择满足条件的元组
  • GROUP BY 子句对WHERE 子句选择出来的元组进行分组
  • HAVING对GROUP BY的分组进行选择
  • SELECT 子句对选择出来的分组产生查询结果

外连接查询

在SQL语句中，外连接在FROM子包中指定，其语法规则如下:

  • 左连接：LEFT [OUTER] JION
  • 右连接：RIGHT [OUTER] JION
  • 全外连接：FULL [OUTER] JION
    连接条件可包含对参与连接的关系进行元组选择的条件
  • 如“表1 Left Outer Join 表2”，则连接后，表1的任何元组t都会出现在结果表中，如表2中有满足连接条件的元组s, 则t与s连接；否则t与空值元组连接；
  • 如“表1 Right Outer Join 表2”，则连接后，表2的任何元组s都会出现在结果表中，如表1中有满足连接条件的元组t, 则t与s连接；否则s与空值元组连接；
  • 如“表1 Full Outer Join 表2”，是前两者的并。
  • 例：查询所有学生姓名及选修课程号为“C01”的成绩，没有选修该课程的学生，成绩显示为空。

视图

  • 创建视图
    一般格式：
    CREATE VIEW <视图名>[（<列名>，<列名>）…] AS <子查询>
  • 删除视图
    格式：
    DROP VIEW <视图名>
  • 删除视图名
    格式：
    DROP VIEW 视图名
  • 更新视图
    插入、删除、修改
    例：将视图STO2中的教师编号S1的教师改名为“王明”
    UPDATE STO2 SET TN='王明' WHERE TNO='S1'
  • 限制条件
    • 如果一个视图是从多个基本表使用联接操作导出的，则此视图不允许更新。
    • 如果在导出视图的过程中，使用了分组和聚合操作，也不允许该视图更新。
    • 若视图的字段来自字段表达式或常量、操作，则不允许对此视图执行INSERT和UPDATE操作，但可以进行DELETE操作。
    • 若视图定义中含有GROUP BY、DISTINCT短语或或嵌套查询，且内层查询的FROM子句中涉及的表也是导出该视图的基本表，则此视图不允许更新。
