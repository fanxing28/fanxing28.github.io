模板语法

Vue 使用一种基于 HTML 的模板语法，使我们能够声明式地将其组件实例的数据绑定到呈现的 DOM 上。所有的 Vue 模板都是语法层面合法的 HTML，可以被符合规范的浏览器和 HTML 解析器解析。
优势：有响应式，减少DOM操作

指令

条件渲染

v-if、v-else、v-else-if

格式：v-if=、v-else-if=、v-else

  • v-if 指令后面添加变量或表达式来判断元素是否显示
  • v-else-if 指令必须和 v-if 相邻

v-show

v-show 的语法和作用和 v-if 几乎相同，都是用来控制元素显示和隐藏的，但是它不能和 v-else 以及 v-else 配合使用，也不能用在 template 上

列表渲染

v-for

循环渲染

格式：v-for=

  • v-for 可以遍历数组和对象（里面 in 和 of 可以换着用）
  • 遍历数组时参数：（item,index）item 可以是解构
  • 遍历对象时参数：（value,key,index）

v-if与v-for一起使用的问题（面试题）

vue2 中 v-for 的优先级更高 vue3 中 v-if 的优先级更高

当 v-if 和 v-for 使用在同一个标签中时：

  • vue2 中：当需要一开始就隐藏时，先通过 v-for 进行循环渲染后在去隐藏，会导致性能的浪费。
  • vue3 中：v-if的优先级更高，所以当判断的条件和 item 有关的时候，会找不到item变量，就会报错
    Vue2和Vue3的共同解决方式：
  • 当判断的条件和 item 有关的时候，先通过获取数据，在计算属性中过滤，在遍历计算属性过滤后的结果
  • 当判断的条件和 item 无关的时候，可以将 v-if 放入当前标签的外层标签，在必要的时候添加 template 标签

v-if与v-show的区别（面试题）

  • v-if 指令会动态地创建或移除 DOM 元素，从而控制元素在页面上的渲染与不渲染；
  • v-show= 指令会动态为元素添加或移除 style="display: none;" 样式，从而控制元素的显示与隐藏；
    两个的使用场景
  • 如果频繁切换，使用 v-show
  • 如果初始状态是隐藏，并且切换频率不高，使用 v-if
    注：总的来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要频繁切换，则使用 v-show 较好；如果在运行时绑定条件很少改变，则 v-if 会更合适。

methods

函数

computed

计算属性：根据现有的 data 等，计算或过滤出一个我们想要的结果

格式：computed: {}

  • 计算属性看似写的是函数，实际上就是这个函数的返回值
  • 计算属性是有缓存的，下一次调用的时候就可以直接使用这个结果
  • 计算属性的值要依赖于响应式数据（可以同时依赖多个响应式数据）
  • 只要依赖的响应式数据发送改变，那么计算属性就会重新求值
    注：
  • 计算属性不能出现异步，必须要立即 return
  • 计算属性是不能被主动修改的（只读），修改计算属性唯一的方式就是修改它的响应式依赖（被动的）
    计算属性和 methods 的使用场景：
  • 事件处理函数一般使用 methods，其他可以优先考虑计算属性
  • Date.now()没有响应式数据

watch

作用：用来对数据进行监听，只要数据改变了，就会触发相应数据变量名的函数

  • 相应的函数可以接受两个参数，分别是改变后的值和改变前的值
  • watch 选项也支持把键设置成用 . 分隔的路径：使用时用 “”
  • 函数里面允许使用异步(常见的有定时器、数据请求)和DOM操作
    深度监听
    默认情况下无法做深度监听
    watch 默认是浅层的：被侦听的属性，仅在被赋新值时，才会触发回调函数——而嵌套属性的变化不会触发。如果想侦听所有嵌套的变更，需要深层侦听器 谨慎使用：
    深度侦听需要遍历被侦听对象中的所有嵌套的属性，当用于大型数据结构时，开销很大。因此请只在必要时才使用它，并且要留意性能。
    Vue3新增的方法 $watch
    作用：让我们可以开启和结束监听
    计算属性vs侦听器
  • 很多情况下，可能计算属性和侦听属性都能用绑定
  • 优先选择计算属性（计算属性缓存，计算属性允许同时依赖多个响应式数据更方便）
  • 只有当出现异步的时候，只能选择侦听器

单向数据流

数据流向只能从父组件流入到子组件，通过props只要修改父组件的值，子组件的props会同步更新在子组件不能去修改props的值

  • 单向数据流的维度是组件和组件之间的单向数据流传递
  • 双向数据绑定的维度是组件内部之间的双向数据流（双向数据绑定是通过v-model将组件中的变量和input里面的内容进行绑定）
    注：可以通过在父组件中的子组件中标签上新增一个自定义事件，它是通过在子组件中利用this.$emit()触发父组件中的事件方法来进行数据的传递。
    如果一定要改变props的值：可以将props的值作为data的初始值

全局API

DOM 更新时机​

当你更改响应式状态后，DOM 会自动更新。然而，你得注意 DOM 的更新并不是同步的。相反，Vue 将缓冲它们直到更新周期的 “下个时机” 以确保无论你进行了多少次状态更改，每个组件都只更新一次。
若要等待一个状态改变后的 DOM 更新完成，你可以使用 nextTick() 这个全局 API：

nextTick()

它表示的是下一次事件轮询的时候再调用，等待页面渲染结束以后再调用

在 vue 改变数据的时候，数据的改变和页面的渲染不是一起发生的(异步) ==> 为了减少 DOM 的操作次数

模板引用 ref

用ref属性，结合 this.$refs.xxx 的方式获取dom节点
注：this.$refs.xxx的方式获取dom节点的时候要注意DOM是否已经在页面渲染了（beforeMount和mounted之间）

  • ref 加在元素上面，可以用于获取dom节点
  • ref 加在组件上面，可以用来获取子组件的实例

生命周期钩子

官方生命周期视图：

生命周期钩子本身也是选项，写在和data并列的位置

(1) 创建阶段

beforeCreate(){}

在beforeCreate的时候，是不能拿到data的数据以及methods的方法等

一般认为不要在beforeCreate里面做数据请求
问题：如果强行在beforeCreate中做数据请求可以吗？
答：可以，因为数据请求是异步操作，它会等待其他选项api执行完才请求数据完成。
beforeCreate(){}和created(){}中间会去初始化选项api

created(){}

已经可以拿到data中的数据了，所以这里可以做数据请求

(2) 挂载阶段

挂载前后，vue自己去初始化创建和插入dom节点

beforeMount(){}

在beforeMount的时候，页面还没有渲染

初始化创建和插入dom节点

mounted(){}

在mounted的时候，页面渲染结束

(3) 更新阶段

只要data的数据发生改变，这两个函数就会执行

beforeUpdate(){}

修改的数据还没有重新渲染到页面

修改的数据开始重新渲染到页面

updated(){}

修改的数据已经重新渲染到页面

注：不能在updated里面修改数据，会造成死循环

因为在updated里面修改数据会在updated、mounted、beforeUpdate构成的圈中进行死循环

(4) 卸载阶段

beforeUnmount(){}

在卸载前需要将一些无法通过unmounted卸载的东西关闭：

  • 如果页面存在定时器，需要在销毁的时候将定时器清掉
  • Websocket长连接，后端可以向前端推送内容,断开长连接
  • addEventListener原生的js事件监听，removeEventListener
  • eCharts或者地图等第三方插件的时候，卸载实例以释放内存

unmounted(){}

unmounted中一般不在操作

注：

  • 创建阶段和挂载阶段的生命周期是会自动调用的
  • 生命周期的执行顺序跟书写顺序无关，只和各阶段名字有关
  • 数据请求可以在created、beforeMount、mounted里面调用

生命周期的各个阶段

在vue实例的整个生命周期的各个阶段，会提供不同的钩子函数以供我们进行不同的操作。先列出vue官网上对各个钩子函数的详细解析。

生命周期钩子 详细
beforeCreate 在实例初始化之后，数据观测(data observer) 和 event/watcher 事件配置之前被调用。
created 实例已经创建完成之后被调用。在这一步，实例已完成以下的配置：数据观测(data observer)，属性和方法的运算， watch/event 事件回调。然而，挂载阶段还没开始，$el 属性目前不可见。
beforeMount 在挂载开始之前被调用：相关的 render 函数首次被调用。
mounted el 被新创建的 vm.el替换，并挂载到实例上去之后调用该钩子。如果root实例挂载了一个文档内元素，当mounted被调用时vm.el 替换，并挂载到实例上去之后调用该钩子。如果 root 实例挂载了一个文档内元素，当 mounted 被调用时 vm.el替换，并挂载到实例上去之后调用该钩子。如果root实例挂载了一个文档内元素，当mounted被调用时vm.el 也在文档内。
beforeUpdate 数据更新时调用，发生在虚拟 DOM 重新渲染和打补丁之前。你可以在这个钩子中进一步地更改状态，这不会触发附加的重渲染过程。
updated 由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。当这个钩子被调用时，组件 DOM 已经更新，所以你现在可以执行依赖于 DOM 的操作。
activated keep-alive 组件激活时调用。
deactivated keep-alive 组件停用时调用。
beforeDestroy 实例销毁之前调用。在这一步，实例仍然完全可用。
destroyed Vue 实例销毁后调用。调用后，Vue 实例指示的所有东西都会解绑定，所有的事件监听器会被移除，所有的子实例也会被销毁。

插槽（内容分发）

1. 概念

为什么要使用插槽？
因为在小的子组件使用template时它相当于outerHTML直接替代整个子组件标签，此时我们需要保留子组件标签内的内容的时候就需要使用到插槽。 简单理解：就是在子组件中使用标签接收父组件中子组件标签上的内容

作用：不仅可以传入js变量，还可以插入标签甚至组件。(和父传子的自定义属性功能非常接近)
注意：谁的template里面就用谁的变量

格式：<slot></slot>
注：数据变量在谁的 template 里面就用谁的变量

2. 插槽默认值

<slot> 标签中间可以写内容，就会作为插槽的默认值（当子组件在父组件中使用中，子组件标签没有书写内容时使用）

3. 具名插槽

解决需求：调用子组件的时候，传入多个元素或者组件，要在子组件的不同位置渲染
使用：在父组件中子组件内的不同标签上添加v-slot指令，子组件中利用标签接收时，添加对应的name属性的属性值
注：必要时给要插入的内容外部包上template标签，并在标签中加上v-slot指令（v-slot可以缩写成 # ）

4. 动态插槽名（了解）

插入的内容在子组件里面的位置进行动态变化

5. 作用域插槽（了解）

作用域插槽的作用：让父组件插入的内容可以访问子组件的作用域
使用作用域插槽：首先可以在slot标签上面添加自定义属性，然后给父组件的子组件标签上添加 v-slot="slotProps"，slotProps是一个对象，变量就在这个对象下面

官网解释：“透传 attribute”指的是传递给一个组件，却没有被该组件声明为 props 或 emits 的 attribute 或者 v-on 事件监听器。最常见的例子就是 class、style 和 id。当一个组件以单个元素为根作渲染时，透传的 attribute 会自动被添加到根元素上。

内置组件

Vue 提供了两个内置组件，可以帮助你制作基于状态变化的过渡和动画：Transition和TransitionGroup

Transition

<Transition> 是一个内置组件，这意味着它在任意别的组件中都可以被使用，无需注册。它可以将进入和离开动画应用到通过默认插槽传递给它的元素或组件上。进入或离开可以由以下的条件之一触发：

  • 由 v-if 所触发的切换
  • 由 v-show 所触发的切换
  • 由特殊元素 <component> 切换的动态组件
