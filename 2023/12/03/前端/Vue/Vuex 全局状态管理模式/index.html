Vuex(全局状态管理模式)
发表于：2023-12-03 | 分类： 前端 Vue

一. 概念与场景

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式 + 库。它采用集中式存储管理应用的所有组件的状态。
解决场景：

  • 多个视图依赖于同一状态。
  • 来自不同视图的行为需要变更同一状态。

    简单解释就是：多个组件使用同一组数据，不同的组件修改同一组数据

二. 安装

  1. 可以通过 https://unpkg.com/vuex@4.0.0/dist/vuex.global.js 这样的方式指定特定的版本。
  2. npm
  3. Yarn

Vuex 通过 Vue 的插件系统将 store 实例从根组件中“注入”到所有的子组件里。且子组件能通过 this.$store 访问到。

其他组件想要使用时，利用this.$store.getters.xxx

(3) mapMutations辅助函数

你可以在组件中使用 this.$store.commit('xxx') 提交 mutation，或者使用 mapMutations 辅助函数将组件中的 methods 映射为 store.commit 调用（需要在根节点注入 store）。

(四) action

(1) 基本使用

Action 类似于 mutation，不同在于：

  • Action 提交的是 mutation，而不是直接变更状态。
  • Action 可以包含任意异步操作。 第二个参数：payload

(五) Module

解决场景：由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。 将 store 分割成 模块（module）。每个模块拥有自己的 state、mutation、action、getter 命名空间：namespaced: true, 优点：使用命名空间可以使模块具有更高的封装度和复用性 命名空间写在与state、getters等同层级中
注：

  • 加了命名空间就需要修改所有使用了辅助函数的地方
  • 没加，只需要修改使用了mapState的地方
